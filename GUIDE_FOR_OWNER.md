## Структура проекта (что где лежит)

```
mezoclub-crm/
│
├── CLAUDE.md                  ← ГЛАВНЫЙ ФАЙЛ. Правила для AI.
│                                 Claude Code читает его автоматически.
│                                 Как конституция проекта.
│
├── GUIDE_FOR_OWNER.md         ← Этот файл. Для тебя.
│
├── .claude/skills/            ← "Рецепты" для Claude Code.
│   ├── create-module/           Как готовые инструкции:
│   ├── build-frontend/          "создай модуль клиентов",
│   └── run-tests/               "сделай фронтенд-страницу"
│
├── migrations/                ← Структура базы данных.
│   └── 001_initial.sql          Как чертёж здания — описывает
│                                 какие таблицы и поля нужны.
│
├── src/                       ← Весь код живёт здесь
│   ├── modules/               ← Бизнес-логика, разбита по темам:
│   │   ├── auth/                 - авторизация (логин/пароль)
│   │   ├── clients/              - клиенты (косметологи, клиники)
│   │   ├── orders/               - заказы
│   │   ├── products/             - товары (AESSOA, GANA FILL...)
│   │   └── dashboard/            - главный экран с цифрами
│   ├── shared/                ← Общие инструменты:
│   │   ├── database/             - подключение к базе данных
│   │   ├── types/                - описание "форм" данных
│   │   └── utils/                - мелкие помощники
│   ├── client/                ← Фронтенд (то что видит пользователь)
│   └── server/                ← Сервер (работает в фоне)
│
├── package.json               ← Список библиотек + команды запуска
├── tsconfig.json              ← Настройки TypeScript
├── .env.example               ← Шаблон секретных ключей
└── docker-compose.yml         ← Запуск БД одной командой
```

## MVP — что мы делаем в первую очередь

Из 30 блоков документа Manus мы берём только 5:

| # | Что | Зачем | Время |
|---|-----|-------|-------|
| 1 | **Авторизация** | Чтобы был логин/пароль и роли | ~6 ч |
| 2 | **Клиенты** | CRUD: создать, посмотреть, изменить, удалить | ~8 ч |
| 3 | **Товары** | Каталог продуктов MezoClub | ~6 ч |
| 4 | **Заказы** | Приём и обработка заказов | ~10 ч |
| 5 | **Dashboard** | Главный экран с цифрами дня | ~8 ч |

**Итого: ~38 часов** (1-2 недели для одного разработчика с Claude Code)

После этого у тебя будет работающая CRM в которую можно заходить,
вносить клиентов, создавать заказы и видеть статистику.

## Как запустить проект

### Шаг 1: Разработчик клонирует репозиторий
```bash
git clone <url> mezoclub-crm
cd mezoclub-crm
```

### Шаг 2: Копирует настройки
```bash
cp .env.example .env
# Открывает .env и вписывает реальные пароли
```

### Шаг 3: Запускает базу данных
```bash
docker-compose up -d
# Поднимает PostgreSQL + Redis в контейнерах
```

### Шаг 4: Запускает Claude Code и начинает работу
```bash
claude
# Claude Code прочитает CLAUDE.md и будет знать все правила
> Создай модуль авторизации по скиллу /create-module auth
```

## Как контролировать разработку (без знания кода)

1. **Git** — каждое изменение фиксируется. Ты можешь видеть историю.
2. **Swagger** — когда API готов, открой http://localhost:3001/api-docs
   Это визуальная страница где видно все API-методы и можно их тестировать.
3. **Dashboard** — когда фронтенд готов, открой http://localhost:3000
   Это то, что будут видеть менеджеры.
4. **Тесты** — `npm test` показывает зелёные галочки если всё работает.
