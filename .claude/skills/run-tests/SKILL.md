# Тесты для модуля: $ARGUMENTS

Создай тесты в `src/modules/$ARGUMENTS/__tests__/`

## Что тестировать

### 1. API тесты (integration) — `$ARGUMENTS.routes.test.ts`
Используй `supertest` для HTTP-запросов к Express app.

```typescript
import request from 'supertest';
import { app } from '../../../server/app';
```

Тесты:
- GET / — возвращает список (200)
- GET /:id — возвращает запись (200) и 404 для несуществующей
- POST / — создаёт запись (201) и 400 при невалидных данных
- PUT /:id — обновляет (200) и 404 для несуществующей
- DELETE /:id — мягкое удаление (200)
- Авторизация: без токена → 401, без прав → 403

### 2. Service тесты (unit) — `$ARGUMENTS.service.test.ts`
Мокай repository, тестируй бизнес-логику.

Для заказов дополнительно:
- Расчёт суммы заказа
- Валидация переходов статусов
- Проверка остатков при создании

## Правила
- Jest + TypeScript
- Каждый тест изолирован (beforeEach очищает состояние)
- Моковые данные реалистичные (украинские имена)
- После создания: запусти `npm test -- --testPathPattern=$ARGUMENTS`
- Все тесты должны быть зелёными
